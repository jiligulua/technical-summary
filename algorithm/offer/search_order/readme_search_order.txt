信手拈来写出完整正确的二叉查找代码，快速排序

在排序的数组（或部分排序的数组）中查找一个数字或统计某一个数字出现的次数，那么都可尝试用二分查找算法。

查找无非下面4种：顺序查找、二分查找、哈希表查找和二叉排序树查找。

哈希表最主要优点：在O(1)时间内查找某一个元素，是效率最高的查找方法。缺点点是：需要额外的空间来实现哈希表。

排序：
插入排序
冒泡排序
归并排序
快速排序
分析它们的优缺点，从1.额外空间消耗；2.平均时间复杂度；3.最差时间复杂度三方面来阐述

C++标准模板库中的map和unordered_map实现了哈希表的功能，而数组是非常简单的哈希表，很多问题阿，都可以用这个简单的哈希表数组来实现的，切记切记阿。那就用array来操作

若数组的大小是一个常数，可认为空间复杂度是O(1)

二叉排序树查找算法对应的数据结构是二叉搜索树
